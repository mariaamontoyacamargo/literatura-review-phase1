Como leer este diagrama
  • Sonnet nunca toca datos directamente — solo decide que tool llamar y con que argumentos
  • Las tools son funciones Python puras: reciben JSON, devuelven JSON, no tienen estado
  • Haiku solo aparece cuando Sonnet llama review o synthesize — el resto es Python sin IA
  • Si una fuente de busqueda falla (sin key, timeout), safe_execute_tool devuelve un objeto de error y el pipeline sigue con las otras fuentes
  • El cache evita re-hacer las mismas busquedas: segunda corrida del mismo pocket es casi gratis

Como leer este diagrama
  • Cada bloque de "Note over S" es una vuelta del loop — Sonnet decide, ejecuta tools, vuelve a pensar
  • El historial crece en cada vuelta: Sonnet recuerda todo lo que paso antes dentro de la sesion
  • Haiku NO ve ese historial — recibe solo el prompt puntual construido en Python para esa tarea
  • Si el coverage score es menor a 0.6, Sonnet vuelve a buscar antes de pasar a review — no avanza hasta tener suficientes papers
  • El ultimo mensaje de Sonnet es siempre un resumen legible: cuantos aceptados, cuantos a revisar, que gaps quedan

Como leer este diagrama
  • Cada capa agrega campos al objeto paper sin borrar los anteriores — al final tienes todo el historial de transformaciones
  • Crossref es la unica llamada externa en la capa de enriquecimiento — solo actua si el paper tiene DOI
  • La deduplicacion ocurre despues del enriquecimiento porque Crossref normaliza los DOIs, lo que mejora la precision del match
  • Un paper puede entrar por NBER y salir con journal, ISSN y citas formales gracias a Crossref — sin buscar de nuevo
  • bid_network.json se construye aparte: lee los JSONs del corpus y construye el grafo de conexiones

4 · Sistema de scoring v2 — escala 0-14

Nuevo sistema inspirado en GRADE y EPPI-Centre Weight of Evidence. 5 dimensiones independientes. Relevancia = 0 sigue siendo RECHAZADO automatico.

Como leer este diagrama
  • D4 relevancia es el unico gate duro: si un paper no responde la pregunta del pocket, no importa que tan bueno sea metodologicamente
  • D1 captura la jerarquia de evidencia: un RCT no es solo "mejor metodologia" — es la unica forma de establecer causalidad limpia
  • D3 es la dimension mas especifica de este proyecto: un paper excelente sobre firmas japonesas puede valer menos que uno modesto sobre MiPyMEs colombianas
  • D2 es aditivo: un paper puede tener venue top Y muchas citas Y estar pre-registrado — cada señal suma independientemente
  • El umbral secundario (score 7 + relevancia=3 + D1 mayor igual 2) existe para no perder papers muy relevantes con evidencia solida aunque no lleguen a 9

Como leer este diagrama
  • pockets.py es el centro del grafo: si quieres cambiar queries o criterios de un pocket, ese es el unico archivo que tocar

Como leer este diagrama

  • El unico costo real es Sonnet (quien decide) y Haiku (quien revisa y sintetiza) — las 3 tools nuevas de enrich, dedup y coverage son completamente gratis
